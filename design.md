# 디자인 시스템 (Design System)

본 문서는 [research.md](./research.md) 사양과 [plan.md](./plan.md) 마일스톤에 따라 UI를 구축할 때 사용하는 **단일 진실 출처**다. 모든 시각·인터랙션 결정은 본 문서를 기준으로 한다.

**대상 사용자**: 중학생(만 13-15세) + 교사 / **플랫폼**: 데스크톱·태블릿 웹 우선 / **구현**: Tailwind CSS + shadcn/ui + Pretendard

---

## 1. 디자인 철학

### 1.1 한 줄로
> **"학생이 주인공이고 AI는 조용한 코치다."**

화려한 AI 연출 대신 학생의 발화와 사고를 화면의 중심에 둔다. AI의 존재감은 명확하되, 시각적 무게는 작아야 한다.

### 1.2 5대 디자인 원칙

1. **Agency First (학생 주체성 우선)** — 학생 발화는 가장 크고, 가장 잘 보이고, 가장 자주 갱신되어야 한다. AI 발화는 차분히 옆에 머문다.
2. **Calm, Not Childish (차분하되 유치하지 않게)** — 중학생은 "어린이 앱"으로 취급받는 것을 거부한다. 채도는 낮추고, 둥근 모서리는 절제 있게.
3. **Clear Channel (채널은 한눈에)** — 팀 모드와 개인 모드는 배경, 라벨, 아이콘이 즉시 구분되어야 한다. 학생이 어디서 말하고 있는지 헷갈리지 않도록.
4. **Quiet by Default (기본은 조용히)** — 알림, 강조, 애니메이션은 절제. 토의 흐름을 끊지 않는다.
5. **Respectful Surface (존중하는 화면)** — 다른 학생의 발화는 항상 동등하게 표시. 다수결 압력을 시각적으로 만들지 않는다.

### 1.3 연결: research.md 제약 ↔ 디자인 결정

| research.md 제약 | 디자인 적용 |
|------------------|------------|
| C1 Agency 보존 | AI 발화는 폭 70% 이하, 학생 발화 풀폭. AI 카드 좌측 5% 액센트 바만 |
| C3 채널 격리 | 개인 모드 배경 = 따뜻한 크림 / 팀 모드 = 차분한 화이트. 절대 혼동 불가 |
| C5 심리적 안전감 | 부정 피드백 = 토스트(노출 짧음), 긍정 피드백 = 카드(지속 노출) |
| C11 학생 발화 보존 | 학생 발화 카드에 인용 보더 + 작성자 닉네임 항시 표시 |
| C12 AI 시각 구분 | AI 발화 전용 아이콘(✦) + 보더 + 미세한 배경 틴트 |

## 2. 컬러 시스템

### 2.1 팔레트 철학
- **Primary**: 신뢰감 있는 청색 계열 (교육 + 집중)
- **AI Accent**: 보라색 계열 (AI 보조 영역과 시각적 분리)
- **Personal Surface**: 따뜻한 앰버/크림 (사적 공간의 안전감)
- **Status**: 의미별 4색 (success / warning / danger / info)
- **Neutral**: 차가운 슬레이트 (배경·텍스트)

채도는 의도적으로 낮춘다. 학습 환경에서 장시간 시청 시 피로를 줄이기 위해.

### 2.7 컬러 사용 규칙

- **AI 발화에는 brand 색상을 쓰지 않는다.** 학생 액션(저장, 공유, 전송)과 AI 출력을 구분하기 위함.
- **danger는 학생 발화 영역에 절대 칠하지 않는다.** 비방 감지 시에도 학생 카드 자체는 변색되지 않고, 별도 토스트로 알림.
- **success는 합의 도달, 결과 저장 등 학생의 성취 순간에만.** 자동 트리거(AI 호출)에는 사용 금지.
- **personal-bg는 개인 모드 채팅 영역에만.** 다른 영역(보드, AI 패널)은 brand 또는 neutral.

### 2.8 컬러 대비 (WCAG)

본문 기준 **AA(4.5:1) 충족**, 중요 텍스트는 **AAA(7:1) 권장**.

| 조합 | 대비 비율 | 등급 |
|------|----------|------|
| `neutral-800` on `neutral-0` | 16.7 : 1 | AAA |
| `neutral-600` on `neutral-0` | 7.5 : 1 | AAA |
| `brand-600` on `neutral-0` | 4.6 : 1 | AA |
| `ai-text` (#5B21B6) on `ai-50` | 9.2 : 1 | AAA |
| `personal-accent` on `personal-bg` | 6.8 : 1 | AA+ |

`brand-500`은 본문 텍스트 사용 금지(대비 부족), 액센트·버튼 배경에만.

## 3. 타이포그래피

### 3.1 폰트 스택

**1순위 (Pretendard Variable)** — 한글·영문·숫자 모두 화면 가독성 최적화.

```css
font-family:
  'Pretendard Variable',
  Pretendard,
  -apple-system,
  BlinkMacSystemFont,
  'Apple SD Gothic Neo',
  'Malgun Gothic',
  'Noto Sans KR',
  sans-serif;
```

코드/수치: `ui-monospace, 'JetBrains Mono', 'D2Coding', monospace`

### 3.2 타입 스케일

**기본 사이즈 17px** (Tailwind 기본 16px보다 +1px) — 중학생 가독성 및 장시간 사용 피로 경감.

| 토큰 | size | line-height | 용도 |
|------|------|-------------|------|
| `text-2xs` | 11px | 16px | 메타 정보 (시각, 상태) |
| `text-xs` | 13px | 18px | 캡션, 라벨 |
| `text-sm` | 15px | 22px | 보조 텍스트, 토스트 |
| `text-base` | **17px** | 26px | **본문 기본** (채팅, 의견 카드) |
| `text-lg` | 19px | 28px | 강조 본문 (AI 안내 첫 줄) |
| `text-xl` | 22px | 30px | 카드 제목 |
| `text-2xl` | 26px | 34px | 패널 헤더 |
| `text-3xl` | 32px | 40px | 페이지 제목 |
| `text-4xl` | 40px | 48px | 인증 페이지 큰 헤더 |

### 3.3 폰트 굵기

| 토큰 | weight | 용도 |
|------|--------|------|
| `font-normal` | 400 | 본문 |
| `font-medium` | 500 | UI 기본 (버튼, 라벨) |
| `font-semibold` | 600 | 강조, 닉네임 |
| `font-bold` | 700 | 제목, 합의 결과 |

**금지**: `font-thin`, `font-light` (300 이하). 한글 가독성 저하.

### 3.4 문장 규칙

- **학생 노출 메시지 1문장 25자 이내** (research.md §2.3 발달 적합성).
- **줄 길이 최대 60-65자** (한글 기준). 채팅 영역 너비를 이 기준으로 설계.
- **letter-spacing**: 기본 `-0.01em` (-1%). 한글에서 자연스러운 시각 무게.
- **한영 혼용 시**: 영문 약어 첫 등장 시 한글 병기. 예: "API(외부 연결 통로)".

### 3.5 단락 스타일

- 본문 단락 사이 간격: `mt-3` (12px).
- 채팅 메시지 사이 간격: 같은 발화자 연속 `mt-1` (4px), 다른 발화자 `mt-4` (16px).
- 제목 → 본문 간격: 제목 line-height 직후 `mt-2` (8px).

## 4. 간격 (Spacing)

### 4.1 스케일

Tailwind 기본 4px 베이스 + 본 프로젝트는 **8의 배수** 우선 사용 (시각적 안정).

| 토큰 | px | 일반 용도 |
|------|----|----------|
| `space-1` | 4 | 아이콘-텍스트 간격 |
| `space-2` | 8 | 미세 간격, 배지 패딩 |
| `space-3` | 12 | 입력창 패딩, 같은 발화자 메시지 간 |
| `space-4` | 16 | **기본 카드 패딩** |
| `space-6` | 24 | 카드 사이, 섹션 내부 |
| `space-8` | 32 | 섹션 간 |
| `space-12` | 48 | 큰 분리 (페이지 헤더 ↔ 본문) |
| `space-16` | 64 | 페이지 상하 여백 (데스크톱) |

### 4.2 패딩 패턴

- **버튼**: `px-4 py-2` (소형) / `px-5 py-2.5` (기본) / `px-6 py-3` (대형 - 결과 저장)
- **입력**: `px-3 py-2.5`
- **카드**: `p-4` (기본), `p-6` (강조 카드 - 합의 영역)
- **다이얼로그**: `p-6`
- **토스트**: `px-4 py-3`

### 4.3 마진 패턴

- 컴포넌트 간 수직: `gap-4` (flex/grid 권장, margin 지양)
- 채팅 메시지 그룹 사이: `mt-6`
- 섹션 헤더 아래: `mb-4`

## 8. 모션 (Motion)

### 8.1 시간 토큰

| 토큰 | duration | easing | 용도 |
|------|----------|--------|------|
| `motion-fast` | 120ms | `cubic-bezier(0.4, 0, 0.2, 1)` | hover, focus, tooltip |
| `motion-base` | 200ms | `cubic-bezier(0.4, 0, 0.2, 1)` | **기본**: 버튼 색 변화, 카드 fade-in |
| `motion-slow` | 320ms | `cubic-bezier(0.32, 0.72, 0, 1)` | 다이얼로그, 사이드 패널 |
| `motion-page` | 480ms | `cubic-bezier(0.32, 0.72, 0, 1)` | 페이지 전환 |

### 8.2 인터랙션 모션

- **메시지 등장**: 새 발화는 위에서 4px slide-down + opacity 0→1 (200ms).
- **AI 발화 등장**: 학생과 동일 모션, 단 50ms 지연. "조용히 등장하는" 느낌.
- **의견 카드 등록**: scale 0.96 → 1, opacity 0 → 1 (240ms).
- **결과 보드 행 추가** (교사): 위에서 slide-down (320ms) + 1초간 `bg-success-50` 강조 후 페이드.
- **합의 도달**: 합의 영역 전체에 success 펄스 1회 (480ms, 1회만).

### 8.3 금지 모션

- **튀어오르는(bounce) 모션**: 토의 흐름을 끊는다. 사용 금지.
- **회전 로딩 스피너 5초 이상**: 진행 막대 또는 텍스트 안내로 대체.
- **자동 캐러셀, 자동 반짝임**: 주의 흐름 방해.
- **`prefers-reduced-motion: reduce`** 사용자에게는 fade만 적용 (slide·scale 제거).

## 10. 채널 모드 시각 구분 (핵심)

### 10.1 팀 모드 vs 개인 모드 비교표

| 요소 | 팀 모드 | 개인 모드 |
|------|---------|----------|
| 채팅 영역 배경 | `bg-neutral-0` (#FFFFFF) | `bg-personal-bg` (#FFFBEB) |
| 채팅 영역 보더 | `border-neutral-200` | `border-personal-border` (#FDE68A) |
| 헤더 배지 | (없음) | 🔒 **"나만 보이는 공간"** (amber-50 bg, amber-700 text) |
| 탭 활성 표시 | brand 인디케이터 | personal-accent 인디케이터 |
| AI 아이콘 | `Sparkles` (violet) | `Sparkles` (violet) — 동일 |
| AI 라벨 | "AI 진행자" | "AI 코치" |
| 다른 학생 노출 | 모든 모둠원 | **없음** (본인만) |
| 메시지 입력창 placeholder | "팀에 의견 남기기…" | "혼자만의 생각을 정리해보세요…" |

### 10.2 모드 전환 인디케이터

- 좌측 채팅 영역 상단 **고정 탭 2개**: `[👥 팀] [🔒 개인]`
- 활성 탭은 보더 하단 2px + 색상 변화.
- 개인 탭으로 전환 시 채팅 영역 배경이 **200ms 페이드**로 personal-bg로 변환.
- 모드 전환 직후 첫 화면 진입 시 **헤더 배지 1초 강조 펄스** (다시 진입 시는 펄스 없음).

### 10.3 "팀에 공유" 액션

- 개인 모드 메시지 옆에 호버 시 `[팀에 공유]` 버튼 등장.
- 클릭 시 확인 다이얼로그: "이 내용을 팀 모두에게 보여줄까요?"
- 공유 후 팀 채팅에 등장하는 메시지는 좌측에 `personal-accent` 4px 액센트 바 + "🔒→👥 개인에서 공유" 캡션.

## 11. AI 발화 시각 구분 (핵심)

### 11.1 AI 메시지 카드 사양

```
┌──────────────────────────────────┐
│ ✦ AI 진행자                       │ ← ai-500 아이콘 + ai-text 라벨 (text-xs)
│ ─────────────────────────────── │
│  발화 본문 (text-base)             │
│  최대 폭: 부모 영역의 70%          │
└──────────────────────────────────┘
배경: ai-50 / 보더: ai-200 (1px) / radius: rounded-lg
```

- **그림자 없음** (보더로 깊이 표현).
- **최대 폭 70%** — 학생 발화(풀폭)보다 좁아 시각적 위계 형성.
- **좌측 정렬** — 학생 발화와 동일 정렬이지만 폭으로 구분.
- 학생이 자신의 발화로 착각하지 않도록 항상 ✦ 아이콘 + "AI 진행자" / "AI 코치" 라벨.

### 11.2 학생 발화 카드 사양

```
┌─────────────────────────────────────────┐
│ 🟣 모둠지기                              │ ← 아바타 + 닉네임 (text-sm semibold)
│ 발화 본문 (text-base)                    │
│ 14:23 · 5번째 발화                       │ ← 메타 (text-2xs neutral-400)
└─────────────────────────────────────────┘
배경: neutral-0 / 보더 없음 (구분선만) / 풀폭
```

본인 발화는 우측 정렬 + brand-50 배경, 다른 학생 발화는 좌측 정렬 + 화이트.

### 11.3 발화자 위계

| 발화자 | 폭 | 정렬 | 배경 | 아이콘 |
|--------|-----|------|------|--------|
| 본인 (학생) | 풀폭 | 우 | brand-50 | (없음 또는 본인 아바타) |
| 다른 학생 | 풀폭 | 좌 | neutral-0 | 닉네임 아바타 |
| AI 진행자 (팀) | 70% | 좌 | ai-50 | ✦ |
| AI 코치 (개인) | 70% | 좌 | ai-50 | ✦ |
| 시스템 (입장/퇴장) | 풀폭 중앙 | 중앙 | (배경 없음) | (없음) |

## 15. 접근성 (Accessibility)

### 15.1 WCAG AA 준수

- 본문 텍스트 대비 ≥ 4.5:1
- 큰 텍스트(18.66px 이상) ≥ 3:1
- 인터랙티브 요소 포커스 링: 2px brand-500 + 2px offset

### 15.2 키보드 네비게이션

- **모든 인터랙티브 요소 Tab 접근 가능**.
- 채팅 영역 내 단축키:
  - `Enter`: 전송
  - `Shift+Enter`: 줄바꿈
  - `Cmd/Ctrl + 1` / `Cmd/Ctrl + 2`: 팀/개인 탭 전환
  - `Esc`: 다이얼로그 닫기
- AI 패널 버튼: 숫자 키 단축키 없음 (오용 방지).

### 15.3 스크린 리더

- AI 메시지: `<article role="status" aria-label="AI 진행자 메시지">`.
- 채팅 메시지: `<article aria-label="{닉네임}의 발화">`.
- 모드 전환: `aria-live="polite"` 영역에 "개인 모드로 전환되었습니다" 안내.

### 15.4 한국어 접근성 특수 사항

- `lang="ko"` 페이지 루트.
- 한자·외국어 혼용 시 `<span lang="en">` 명시.
- TTS가 자연스럽도록 약어 풀어쓰기 (`AI` → 발음 노출, `API` → "API(외부 연결 통로)").

### 15.5 모션 감소

`@media (prefers-reduced-motion: reduce)`:
- slide, scale, bounce 제거
- opacity transition만 유지 (120ms)
- 자동 펄스 모두 제거

### 16.1 브레이크포인트

### 16.3 반응형 변화 요약

| 영역 | xl ≥ 1280px | md 768-1024 | sm < 640 |
