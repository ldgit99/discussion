# 파일럿 테스트 준비

[plan.md §4.4](../plan.md) 4주차 산출 외 항목 정리.

## 1. 보호자 동의 — research.md §8.1

연구·교육 목적이 아닌 일반 운영이라도 만 14세 미만이 포함될 가능성이 있으면 보호자 동의 필요.

### 동의서 권장 항목

- 서비스 명칭 + 목적 (중학생 토의 보조 AI)
- 수집 항목:
  - 회원가입: 이메일, 반/번호/이름 (닉네임 생성용)
  - 토의 활동: 메시지·의견 카드 텍스트 (수업 운영 목적)
- 보관 기간: 학기 종료 후 N개월 이내 삭제
- 제3자 제공:
  - OpenAI (메시지 텍스트가 모델 입력으로 전송됨, 학습에는 사용 안 됨 — OpenAI Data Processing Addendum 명시)
  - Supabase (한국 사용자는 ap-southeast-1 싱가포르 리전, GDPR 준수)
- 학생·보호자 권리: 열람·삭제 요청 방법
- 동의 철회 절차

> 학교·교육청 표준 양식이 있으면 그것을 우선 사용하고, 위 항목을 추가하세요.

## 2. 사전·사후 설문 — research.md §7

파일럿 테스트 후 성과 측정용. [research.md §7.3](../research.md) Agency 보존 지표가 핵심.

### 사전 설문 (5분)
1. 평소 모둠 토의에서 의견을 잘 말하는 편인가요? (1-5)
2. 친구들의 의견을 끝까지 듣는 편인가요? (1-5)
3. AI 도구를 수업에서 써본 경험이 있나요? (예/아니오)

### 사후 설문 (10분)
4. 이번 토의에서 평소보다 더 많이 말했나요? (1-5)
5. AI가 내 생각을 대신했다고 느꼈나요? (1-5, 5=강하게 그렇다)
   - **목표: 평균 2.5 이하**
6. 모둠 친구의 의견을 더 잘 이해하게 됐나요? (1-5)
7. 가장 도움이 된 AI 기능 (택1):
   - 정리하기 / 비교 / 근거 확인 / 질문 / 합의 돕기 / 태도 점검 / 개인 코치
8. 자유 의견 (한 줄)

권장 도구: Google Forms 또는 자체 페이지.

## 4. 운영자 대응 매뉴얼

### AI 응답 지연 (>5초)
- 학생 화면에 "AI가 생각 중이에요…" 표시
- 10초 초과 시 자동 재시도 1회 → 그래도 안 되면 "잠시 후 다시 눌러주세요"

### 부적절 표현 다발
- attitude_flags 테이블에 누적 기록 (교사 대시보드 표시)
- 교사가 직접 학생 상담

### Supabase 연결 실패
- 학생 화면에 "연결이 끊겼어요" 배너
- 자동 재연결 (Supabase Realtime 기본 동작)
- 5분 이상 지속 시 교사가 종이 토의로 전환

### 학생이 같은 닉네임으로 가입 시도
- 같은 반/번호가 이미 있으면 가입 단계에서 차단됨 (운영 단계 추가 검증)
- 임시: 한 학생이 이미 가입했다면 그 계정으로 로그인 안내
